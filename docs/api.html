API¶

Local¶

The code that should be run locally

travis_docs_builder.local.encrypt_variable(variable, repo, public_key=None)[source]¶

Encrypt an environment variable for repo for Travis

variable should be a bytes object, of the form b'ENV=value'.

repo should be like ‘gforsyth/travis_docs_builder’.

public_key should be a pem format public key, obtained from Travis if not provided.

travis_docs_builder.local.generate_GitHub_token(username, password=None, OTP=None, note=None, headers=None)[source]¶

Generate a GitHub token for pushing from Travis

The scope requested is public_repo.

If no password or OTP are provided, they will be requested from the command line.

The token created here can be revoked at https://github.com/settings/tokens. The default note is “travis_docs_builder token for pushing to gh-pages from Travis”.

Travis¶

The code that should be run on Travis

travis_docs_builder.travis.commit_docs(*, built_docs='docs/_build/html', gh_pages_docs='docs', tmp_dir='_docs')[source]¶

Commit the docs to gh-pages

Assumes that setup_GitHub_push(), which sets up the origin_token remote, has been run and returned True.

travis_docs_builder.travis.create_gh_pages()[source]¶

If there is no remote gh-pages branch, create one.

Return True if gh-pages was created, False if not.

travis_docs_builder.travis.get_token()[source]¶

Get the encrypted GitHub token in Travis

Make sure the contents this variable do not link. The run() function will remove this from the output, so always use it.

travis_docs_builder.travis.gh_pages_exists()[source]¶

Check if there is a remote gh-pages branch.

This isn’t completely robust. If there are multiple remotes and you have a gh-pages branch on the non-default remote, this won’t see it.

travis_docs_builder.travis.push_docs()[source]¶

Push the changes to the gh-pages branch.

Assumes that setup_GitHub_push() has been run and returned True, and that commit_docs() has been run. Does not push anything if no changes were made.

travis_docs_builder.travis.run(args)[source]¶

Run the command args

Automatically hides the secret GitHub token from the output.

travis_docs_builder.travis.setup_GitHub_push(repo)[source]¶

Setup the remote to push to GitHub (to be run on Travis).

This sets up the remote with the token and checks out the gh-pages branch.

The token to push to GitHub is assumed to be in the GH_TOKEN environment variable.
